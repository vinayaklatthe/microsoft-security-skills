Removing standing Global Admin and moving to just-in-time elevation is exactly what Entra ID Privileged Identity Management (PIM) is for. Here is the approach.

What "standing access" means and why to remove it
- Standing access = an account is permanently assigned Global Administrator (or another privileged role). If that account is phished, the attacker immediately has full control.
- Just-in-time (JIT) = the account is only eligible for the role. It holds no privilege day to day and must activate the role for a limited time, with MFA and (ideally) approval, before it can act.

Steps to implement

1. Inventory current privileged assignments
- List everyone with Global Administrator and other high-impact roles (Privileged Role Admin, Security Admin, Exchange/SharePoint Admin, Application Admin, etc.).
- In the portal: Entra ID > Roles and administrators, or PIM > Roles.

2. Convert active assignments to eligible
- In PIM, for each privileged role, change permanent/active assignments to "Eligible".
- Keep only 2 break-glass / emergency accounts as permanent Global Admins, excluded from the JIT process and from Conditional Access, with FIDO2 keys, and monitored.
- Aim to keep total Global Admins small (under 5) and prefer least-privilege roles over Global Admin where possible.

3. Configure activation settings per role (role settings in PIM)
- Maximum activation duration (e.g. 1-4 hours, not 8+).
- Require MFA on activation (phishing-resistant for the most sensitive roles).
- Require justification, and for Global Admin require approval by a designated approver.
- Require ticket information if you integrate with ITSM.
- Turn on notifications for activation and assignment changes.

4. Add governance
- Set eligible assignments to expire (time-bound) rather than permanent eligibility, and require periodic renewal.
- Configure access reviews on privileged roles (e.g. quarterly) so stale eligibility is removed.
- Use PIM for Groups if you manage privilege through role-assignable groups.

5. Protect the activation path
- Require phishing-resistant MFA via Conditional Access for anyone activating admin roles.
- Use dedicated cloud-only admin accounts, ideally from secured/privileged workstations.

6. Validate
- Confirm break-glass accounts still work.
- Test that an admin can activate a role, perform the task, and that it deactivates automatically.
- Review the PIM audit logs to confirm activations are logged.

End state: nobody walks around with permanent Global Admin (except tightly controlled break-glass). Admins activate the specific role they need, for a short window, with MFA and approval, and everything is logged and reviewed. This dramatically shrinks the window an attacker can use stolen credentials. Note PIM requires Entra ID P2 licensing.
